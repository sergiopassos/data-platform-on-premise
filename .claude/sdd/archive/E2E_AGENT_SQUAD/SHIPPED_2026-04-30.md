## Summary

A 6-agent autonomous squad built with LangGraph `StateGraph` that executes a full Medallion E2E test against the KIND Kubernetes data platform:

**Postgres CDC → Kafka → Bronze Iceberg → Silver Spark MERGE → Gold dbt → Trino COUNT**

The squad runs from a single CLI command (`python -m agents.run_e2e --table <name>`), auto-starts all required `kubectl port-forward` tunnels, emits Langfuse observability scores, and produces a Markdown RCA report on any failure.

## What Was Built

| Component | Key Decisions |
|-----------|--------------|
| `agents/state.py` | `E2EState` TypedDict with `last_completed` (not in DESIGN) to enable orchestrator hub-and-spoke routing without agent coupling |
| `agents/orchestrator/router.py` | Purely deterministic — indexes into `_PIPELINE` list using `last_completed`; Fail-Fast in one place |
| `agents/graph.py` | All agents return to orchestrator; `reporter` → END; `build_graph()` returns compiled graph |
| `agents/port_forward.py` | `PortForwardManager` — 8 `kubectl port-forward` tunnels via `subprocess.Popen` + 3s warmup + `atexit` cleanup |
| `agents/run_e2e.py` | `--no-port-forward` flag for environments where tunnels are already active |
| `agents/infrastructure/` | Switched from jsonpath `\n`-delimited parsing to `-o json` + JSON parse for reliable pod readiness detection (caught a real bug on first real run) |
| `agents/data_source/` | `asyncio.run()` isolates async `ODCSGenerator.generate()` inside synchronous LangGraph node; Debezium 409 = idempotent success |
| `agents/spark_processing/` | Python `str.replace()` renders Jinja2-style template for silver manifest; Nessie REST for table existence verification |
| `agents/gold/` | 1-retry pattern on dbt DAG failure; COUNT=0 → ERROR |
| `agents/reporter/` | Always runs (Fail-Fast path and success path); late-imports Langfuse to avoid crash when unconfigured |
| `pyproject.toml` | `setuptools.build_meta` backend (not `setuptools.backends.legacy` — incompatible with setuptools < 69 / Python 3.13) |

## Deviations from DESIGN

| Item | DESIGN | Shipped | Reason |
|------|--------|---------|--------|
| `last_completed` field | Not in state schema | Added | Orchestrator needed to know which agent just ran to index into `_PIPELINE` |
| `gold_table_fqn` field | Not in state schema | Added | dbt gold models are static (`customers_orders`, `orders_summary`); configurable via `--gold-table` |
| `PortForwardManager` | Not in DESIGN | Added | Post-build user request: CLI must be self-contained, no manual port-forward prerequisite |
| `check_namespace_pods` | jsonpath `\n` parsing | JSON `-o json` parsing | jsonpath delimiter produced concatenated output on real cluster — caught on first run |
| `pyproject.toml` build backend | `setuptools.backends.legacy:build` | `setuptools.build_meta` | `legacy` backend incompatible with Python 3.13 environment |
| `psycopg[binary]` pin | `==3.1.18` | `>=3.2.2` | Python 3.13 requires psycopg ≥ 3.2 |

## Lessons Learned

### What Went Well

| Category | Lesson |
|----------|--------|
| **Process** | The 5-phase SDD flow (Brainstorm → Define → Design → Build → Ship) eliminated ambiguity before a line of code was written. The DESIGN's file manifest made the build a pure execution task. |
| **Technical** | `last_completed` field in state was the right call — the DESIGN was ambiguous on routing advancement. Deviating early and documenting it avoided a subtle runtime bug. |
| **Technical** | Reusing `portal/agent/` imports (ConnectorActivator, ODCSGenerator, PostgresSchemaInspector) saved ~200 lines of code and kept the agent thin over tested logic. |
| **Technical** | `asyncio.run()` isolation for the single async LangGraph node was cleaner than converting the entire graph to async. One import, one line. |
| **Testing** | Writing unit tests that mock at the agent boundary (not the tool boundary) kept tests fast and meaningful — 46 tests in 0.7s. |
| **Architecture** | Hub-and-spoke orchestrator (all agents → orchestrator → next) means Fail-Fast lives in exactly one place and is trivially testable. |

### What to Improve

| Category | Lesson |
|----------|--------|
| **Technical** | `jsonpath` `\n` delimiter is unreliable across kubectl versions — always use `-o json` + Python JSON parsing for multi-item kubectl output. |
| **Technical** | Build backend choice matters for newer Python versions. Pin `setuptools.build_meta` explicitly in new projects; `setuptools.backends.legacy` is not universally available. |
| **Technical** | `psycopg` version pins should use `>=X.Y` not `==X.Y` when crossing Python major versions — patch-level compatibility is tight. |
| **Process** | The `PortForwardManager` feature was added post-build during the real-cluster test run. It should have been in the DESIGN (it's a usability prerequisite, not an optimization). Add "can the CLI run standalone?" as a DEFINE question for future test-harness features. |
| **Testing** | The jsonpath parsing bug was only caught on a real cluster run — unit tests mocked the subprocess output. Consider adding a "schema validation" test that asserts the parsed pod dict has the expected keys. |

## Known Operational Constraints

- Spark image `data-platform/spark:3.5.1` must be pre-loaded into KIND nodes: `kind load docker-image data-platform/spark:3.5.1 --name data-platform`
- `GEMINI_API_KEY` or Ollama for LLM contract generation — falls back to deterministic generator if neither is present
- Gold DAG (`gold_dbt_dag`) sources from `silver.customers` and `silver.orders` — use `--table customers` or `--table orders` for a meaningful E2E run
- `LANGFUSE_PUBLIC_KEY` / `LANGFUSE_SECRET_KEY` are optional — Reporter degrades gracefully to state-only report if absent
- Run from repo root with the virtualenv active: `python -m agents.run_e2e --table customers`
